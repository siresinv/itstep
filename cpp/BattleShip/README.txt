ФИШКИ ИГРЫ
 1. Есть меню
 2. Исключен некорректный выбор пунктов меню
 3. Выбор типа игры
 4. Приостанавка игры. Возврат в игру
 5. Просмотр статистики игры, игроков
 6. Перезапуск игры - поля остаются, статистика игроков обнуляется
 7. Удаление игры
 8. Создание новой игры
 9. Программный выход из игры


ФИШКИ ИГРОВОГО ПРОЦЕССА
 1. Каждую новую игру корректная рандомная расстановка кораблей
 2. Ход запрашивается заново, если некорректно выстрелил (повторно по клетке)
 3. Обнаружение, что корабль потоплен.
 4. Расстановка вокруг потопленного корабля спецсимволов,
	при стребльбе по которым, ход тоже будут считаться некорректным
 5. Исключен некорректный ввод клетки
 6. Если есть попадание, то ход не переходит (а надо ли так?)
 7. Если один из игроков выиграл, то у другого есть возможность доиграть


ФИШКИ КОДА
 1. Игра и игроки создаются как структуры
 2. Игра содержит массив структур игроков
 3. Структуры игры и игроков содержат множество параметров
	проверяемых и изменяемых в процессе игры
 4. Универсальность - при минимальных изменениях в коде размер полей,
	количества кораблей и игроков может быть любым и неограниченным
	- но кому это надо). Но для отладки - очень даже удобно
 5. Так же минимальными изменениями в коде для отладки можно задать -
	видит ли игрок поле противника или нет
